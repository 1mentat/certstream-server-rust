## Human Verification

| Criterion | Justification | Verification Approach |
|-----------|--------------|----------------------|
| migrate-streaming.AC1.1 | Confirming that the production code calls `execute_stream()` rather than `collect()` is a structural property that cannot be verified by a black-box functional test alone. The automated tests prove correctness of output, but not that the specific API was used. | Code review: verify that `partition_df.execute_stream().await` replaced `partition_df.collect().await` in `run_migrate` (around line 1278 of `src/backfill.rs`). Confirm that no `.collect()` call remains in the per-partition data path. Confirm that `use futures::stream::StreamExt;` is present in imports. |
| migrate-streaming.AC1.2 | Proving that no accumulation occurs is a memory-behavior property. Automated tests verify correct output but cannot directly observe that batches are not held in memory simultaneously. The implementation structure (stream loop with per-iteration write and implicit drop) provides this guarantee, but it requires code inspection to confirm. | Code review: verify the per-partition loop uses `while let Some(batch_result) = stream.next().await` and that the batch variable is consumed by the `DeltaOps.write()` call within the same loop iteration. Confirm there is no `Vec` or other collection accumulating batches across iterations. Each `batch` goes out of scope (is dropped) at the end of each loop iteration before `stream.next().await` fetches the next one. |
| migrate-streaming.AC1.3 | The specific error path for a mid-stream batch read failure (where `execute_stream()` succeeds but a subsequent `stream.next().await` returns `Some(Err(...))`) is difficult to trigger in a unit test without mocking DataFusion internals. The code path is straightforward (`match batch_result { Err(e) => return 1 }`), making code review sufficient. | Code review: verify the `while let Some(batch_result) = stream.next().await` loop contains a `match batch_result` with an `Err(e)` arm that logs a warning and returns exit code 1. The existing `test_migrate_nonexistent_source_table` test covers the stream-creation error path (where `execute_stream()` itself fails). |
